# Requirements for Outgoing Repositories

Thank you for your time maintaining a project under Django Commons!

To see how to initiate a transfer out of Django Commons, see [How to transfer a project out?](https://django-commons.org/#how-to-transfer-a-project-out)

There are a few requirements that must be met before a project can be transferred out:

- All current maintainers and repository admins agree to the transfer.
- A justification has been formed to transfer the project.
- The receiving owner (individual or organization) is identified.

## Solo Admin

If a project has only one admin and that admin is new to the project (i.e., they were added by Django Commons rather than being an original maintainer), the Django Commons admins team will review whether a transfer out is appropriate before approving the request. A new admin must have been in the role for at least 6 months and had a minimum of one release before a transfer out can be initiated.

## Infrastructure

If the project has any active funding, DNS, or hosting arrangements under Django Commons, those should be formally closed or transferred before the repository leaves. The outgoing maintainer is responsible for communicating with any relevant parties about the change.
